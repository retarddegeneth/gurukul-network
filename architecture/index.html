Four operational circuits

Each is a distinct read-write flow — enrollment, verification, guru operations, and disputes.

Enrollment flow
  1. 1Student (wallet) → /marketplace → picks Guru
  2. 2→ Enrolls; escrow funded on Robinhood chain
  3. 3→ AI Registrar seals exam items (VRF)
  4. 4→ AI Dean (Guru) delivers Teaching → Training → Production
  5. 5→ CoE evaluator grades → mint credential to student
  6. 6→ Comptroller settles: escrow → Guru minus protocol fee → treasury
Verification flow (recruiter/public)
  1. 1Recruiter → /address/{wallet} → sees all credentials for that address
  2. 2→ Click credential → /credential/{id} → view on-chain proof + delta
  3. 3→ Explorer link → verify tx on Robinhood explorer
  4. 4Independent · needs no cooperation from student or Guru
Guru operator flow
  1. 1Operator (wallet) → /guru/register → stakes ETH → publishes profile
  2. 2→ Guru appears on /marketplace
  3. 3→ Sessions land in /guru/dashboard
  4. 4→ Earnings visible per-session; withdraw stake or claim payouts
  5. 5→ Reputation is aggregate verified delta of students
Dispute / slashing flow
  1. 1Anyone → Grievance desk (Layer 3 pod)
  2. 2→ AI Registrar cross-checks item-bank + transcript
  3. 3→ CoE evaluator committee votes
  4. 4→ On confirmed fraud: revokeCredential() + slash Guru stake
  5. 5→ Comptroller redirects slashed stake → treasury / affected student
